# TBFusionAI Quick Reference

## 🚨 Current Issue

**Problem:** Docker container is running but predictions fail because models aren't trained.

**Root Cause:** The ML pipeline hasn't been executed yet - no data downloaded, no features extracted, no models trained.

**Solution:** Run the pipeline inside the container.

---

## ⚡ Quick Fix (Choose One)

### Option A: Run Pipeline in Docker (Recommended)

```bash
# Start the container (if not running)
docker-compose -f docker/docker-compose.yml up -d

# Run the complete pipeline (~2 hours)
docker exec tbfusionai-api python main.py run-pipeline

# Monitor progress
docker-compose -f docker/docker-compose.yml logs -f

# Check when done
docker exec tbfusionai-api python main.py status
```

### Option B: Run Pipeline Locally

```bash
# Stop Docker container
docker-compose -f docker/docker-compose.yml down

# Run pipeline locally
python main.py run-pipeline

# Start Docker again (models will be in mounted volume)
docker-compose -f docker/docker-compose.yml up -d
```

## 🔍 Troubleshooting

### Container Keeps Showing "Models Not Found"

```bash
# Check if pipeline has run
docker exec tbfusionai-api python main.py status

# If all stages show "Pending", run:
docker exec tbfusionai-api python main.py run-pipeline
```

### Pipeline Seems Stuck

```bash
# Check logs for progress
docker-compose -f docker/docker-compose.yml logs -f

# Audio processing stage is SLOW (60+ min) - this is normal
# Look for progress bars in logs
```

### Out of Memory Error

```bash
# Increase Docker memory limit to 8GB+
# Edit Docker Desktop settings → Resources → Memory

# Or run locally with:
python main.py run-pipeline
```

### Want to Start Over

```bash
# Stop container
docker-compose -f docker/docker-compose.yml down

# Remove artifacts (WARNING: deletes all data & models)
rm -rf artifacts/

# Rebuild and restart
docker-compose -f docker/docker-compose.yml build --no-cache
docker-compose -f docker/docker-compose.yml up -d

# Run pipeline from scratch
docker exec tbfusionai-api python main.py run-pipeline --force
```

## 💡 Pro Tips

1. **First Setup:** Run overnight - takes 2 hours
2. **Development:** Use local setup for faster iteration
3. **Production:** Use Docker with volume mounts for data persistence
4. **Debugging:** Check `artifacts/logs/tbfusionai.log` for details
5. **Monitoring:** Health endpoint updates every 30 seconds

## 🆘 Still Having Issues?

1. **Check logs:** `docker-compose -f docker/docker-compose.yml logs -f`
2. **Check status:** `docker exec tbfusionai-api python main.py status`
3. **Check disk space:** `df -h` (needs ~10GB free)
4. **Check memory:** Docker should have 8GB+ RAM allocated
5. **Try force re-run:** `docker exec tbfusionai-api python main.py run-pipeline --force`

## 📞 Commands Cheat Sheet

```bash
# Status
docker exec tbfusionai-api python main.py status
docker exec tbfusionai-api python main.py info

# Run Pipeline
docker exec tbfusionai-api python main.py run-pipeline

# Individual Stages

docker exec tbfusionai-api python main.py evaluate-models

docker-compose -f docker/docker-compose.yml logs -f    # Logs

# Clean Start
docker-compose -f docker/docker-compose.yml down
docker-compose -f docker/docker-compose.yml build --no-cache
docker-compose -f docker/docker-compose.yml up -d
```
